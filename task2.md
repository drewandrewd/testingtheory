# Баг-репорты
## Баг Репорт UI-1: Поиск не находит файлы, находящиеся в корзине

**Компонент:** UI - Поиск  
**Окружение:**
- Платформа: Web (Яндекс.Диск)
- Браузер: Firefox
- ОС: Windows 11

### Предусловия
1. На Яндекс.Диске существует файл с уникальным именем (нефайл.txt)
2. Указанный файл удалён и находится в разделе Корзина

### Шаги воспроизведения
1. Открыть веб-интерфейс Яндекс.Диска
2. В поле поиска ввести имя файла (или уникальную часть имени).
3. Выполнить поиск.

### Фактический результат
Файл, находящийся в корзине, не отображается в результатах поиска

### Ожидаемый результат
Согласно требованиям:
> Если файл находится в корзине — он также может быть найден поиском.

Файл из корзины должен отображаться в результатах поиска.

### Серьёзность
**Major**

### Приоритет
**High**

### Примечания
Фактическое поведение системы не соответствует заявленным требованиям.

## Баг Репорт UI-2: Некорректный текст сообщения при отсутствии результатов поиска

**Компонент:** UI - Поиск  
**Окружение:**
- Платформа: Web (Яндекс.Диск)
- Браузер: Firefox
- ОС: Windows 11

### Предусловия
На диске отсутствуют файлы и папки, удовлетворяющие поисковому запросу

### Шаги воспроизведения
1. Открыть веб-интерфейс Яндекс.Диска
2. Ввести в поле поиска заведомо отсутствующий запрос (zolotoy)
3. Выполнить поиск

### Фактический результат
Отображается сообщение:  
**Ничего не нашлось.**

### Ожидаемый результат
Согласно требованиям, должно отображаться сообщение:  
**«В вашем Диске не найдено файлов или папок по запросу `<запрос>`»**

### Серьёзность
**Minor**

### Приоритет
**Medium**

### Примечания
Несоответствие текста сообщения требованиям может приводить к расхождениям UX и автотестов.

## Баг Репорт API-1: Успешное создание папки возвращает HTTP 201 вместо ожидаемого 200

**Компонент:** API 
**Метод:** PUT /v1/disk/resources

### Предусловия
- Используется валидный OAuth-токен
- Папка по указанному пути отсутствует

### Шаги воспроизведения
1. Отправить запрос:  
   `PUT /v1/disk/resources?path=test_folder`  
   `Authorization: OAuth <валидный_токен>`  
   `Content-Type: application/json`
2. Получить ответ сервера.

### Фактический результат
- HTTP статус: 201 Created
- Тело ответа:
  ```json
  {
    "method": "GET",
    "href": "https://cloud-api.yandex.net/v1/disk/resources?path=disk:/test_folder",
    "templated": false
  }
- Папка успешно создана

### Ожидаемый результат
Согласно требованиям:  
- HTTP статус: 200 OK
- Папка успешно создана

### Серьёзность
**Minor**

### Приоритет
**Medium**

### Примечания
Несоответствие текста сообщения требованиям может приводить к расхождениям UX и автотестов.

## Баг Репорт API-2: При создании папки по существующему пути возвращается HTTP 409 вместо 400

**Компонент:** API
**Метод:** PUT /v1/disk/resources

### Предусловия
- Используется валидный OAuth-токен
- Папка по указанному пути уже существует (test)

### Шаги воспроизведения
1. Отправить запрос:  
   `PUT /v1/disk/resources?path=test_folder`  
   `Authorization: OAuth <валидный_токен>`  
   `Content-Type: application/json`
2. Получить ответ сервера

### Фактический результат
- HTTP статус: 409 Conflict
- Тело ответа:
  ```json
  {
  "error": "DiskPathPointsToExistentDirectoryError",
  "description": "Specified path points to existent directory.",
  "message": "По указанному пути уже существует папка с таким именем."
  }

### Ожидаемый результат
Согласно требованиям:
- HTTP статус: 400 Bad Request
- Папка не должна быть создана.

### Серьёзность
**Minor**

### Приоритет
**Medium**

### Примечания
Код ответа противоречит требованиям
